选择正确的Minecraft服务器核心（服务端）是成功::material-symbols:terminal::构建服务器的第一步。不同的服务端软件各有特色，适用于不同场景和需求。本指南将帮助你了解并选择最适合你的服务器核心，帮你踏出正确的第一步，拥有一个良好开端。

## 什么是服务器核心？

### 基础概念

**服务器核心**就是==让Minecraft能够以多人游戏模式运行的程序文件==。如果把Minecraft服务器比作一栋房子，那么服务器核心就是这栋房子的地基和框架。

简单来说就是：
- 你平时玩的Minecraft客户端 = 单人游戏
- 服务器核心 = 让多人可以同时进入同一个世界游戏的程序

### 核心的本质

服务器核心通常是一个 `.jar` 文件（Java程序），它包含了：
- 游戏世界的运算逻辑
- 玩家连接管理
- 方块、生物、物品等游戏元素的处理
- 指令系统
- 插件/模组接口 <Badge type="info" text="部分核心" />

### 为什么我需要选择核心？

如果你没接触过架设MC服务器，你可能会问，为什么我需要在服务器核心这一方面做选择？难道官方的服务器核心不好吗？

Mojang官方确实提供了服务器核心（Vanilla），但==它的功能有限=={.warning}，因此Minecraft社区开发了各种各样的“增强版”核心：
- 更好的性能优化
- 插件支持(添加各种功能)
- 模组支持(改变游戏玩法)
- 反作弊功能
- 更多管理工具

!!难不成你还想用这核心？!!

## 如何选择合适的核心

### 决策流程图

```
想开什么类型的服务器?
│
├─ 纯原版生存 → Vanilla 或 Paper(优化更好)
│
├─ 插件服(小游戏、RPG、生存辅助等)
│   │
│   ├─ 小型服务器(10人以下) → Spigot
│   │
│   └─ 中大型服务器 → Paper(推荐) 或 Purpur
│
├─ 模组服(科技、魔法、冒险等)
│   │
│   ├─ 大型整合包 → Forge
│   │
│   └─ 轻量模组/新版本 → Fabric
│
└─ 插件+模组混合 → Mohist/Arclight(谨慎使用)
```

::: note
如果你比比较熟悉这些核心，那么你完全可以自行决定，不必死板地按照上面的决策树来选择你的服务器核心
:::

### 选择建议

**新手服主优先选择顺序**：

::: steps
1. **Paper** - 适合90%的插件服需求
2. **Fabric** - 轻量模组服首选
3. **Forge** - 大型整合包必选
4. **Purpur** - ~~高手~~服主，需要更多自定义
5. **Spigot** - Paper的备选方案
:::

==**不要选择以下核心！**=={.warning}
- 混合核心（除非你很清楚自己在做什么）
- 已停更的核心
- 来源不明的核心 <Badge type="danger" text="不推荐" />

## 核心的下载与安装

### 准备工作

::: steps

1. **确认游戏版本**

   首先确定你要开的服务器版本（如1.20.1），==核心版本必须与客户端版本一致=={.important}。

2. **安装Java**

   不同Minecraft版本需要不同的Java版本：

   | Minecraft版本 | 所需Java版本 |
   |--------------|-------------|
   | 1.12-1.16.5  | Java 8      |
   | 1.17-1.17.1  | Java 16     |
   | 1.18-1.20.4  | Java 17     |
   | 1.20.5-1.21.11 | Java 21     |
   | 26.1+ | Java 25 |

   **Java下载**：
   - 你需要下载对应版本的JDK（Java Development Kit），这里给出两个选项：
   - Adoptium（OpenJDK）：[https://adoptium.net/](https://adoptium.net/)
   - Oracle（甲骨文）：[https://www.oracle.com/cn/java/technologies/downloads/](https://www.oracle.com/cn/java/technologies/downloads/)

3. **使用检查Java版本的命令行检测版本**：
   ```bash
   java -version
   ```

:::

---

### 下载核心

::: steps

1. **Paper下载**

   1. 访问官网：[https://papermc.io/downloads](https://papermc.io/downloads)
   2. 选择游戏版本(如1.20.1)
   3. 点击下载最新构建版本
   4. 得到文件如：`paper-1.20.1-196.jar`

2. **Fabric下载**

   1. 访问官网：[https://fabricmc.net/use/server/](https://fabricmc.net/use/server/)
   2. 选择游戏版本
   3. 下载Fabric服务端安装器
   4. 运行安装器,选择安装路径
   5. 生成服务端文件

3. **Forge下载**

   1. 访问：[https://files.minecraftforge.net/](https://files.minecraftforge.net/)
   2. 选择Minecraft版本
   3. 下载Installer(安装器)
   4. 运行安装器:
      ```bash
      java -jar forge-installer.jar --installServer
      ```
   5. 等待安装完成

4. **更多核心下载链接可以自行搜索……**

:::

### 安装步骤

对于Paper/Spigot等一类的服务器核心，你可以直接用启动脚本运行：

> 详情见：[启动脚本入门](/guide/ji-chu-zhi-shi-xue-xi/startup/)

::: steps

1. **创建服务器文件夹**：
   ```
   新建文件夹，如：MyServer
   ```

2. **放入核心文件**：
   ```
   将下载的 .jar 文件放入文件夹
   重命名为 server.jar (可选，便于管理)
   ```

3. **首次启动**：
   
   Windows系统创建 `start.bat`：
   ```batch title="start.bat"
   @echo off
   java -Xms2G -Xmx4G -jar server.jar nogui
   pause
   ```
   
   Linux系统创建 `start.sh`：
   ```bash title="start.sh"
   #!/bin/bash
   java -Xms2G -Xmx4G -jar server.jar nogui
   ```

4. **同意EULA**：
   
   首次运行会生成 `eula.txt`，打开文件：
   ```plain  title="eula.txt"
   eula=false（将false改为true）
   ```

5. **再次启动**：
   
   运行启动脚本，服务器开始生成世界

:::

---

#### 参数说明

```bash
java -Xms2G -Xmx4G -jar server.jar nogui
```

- `java`：调用Java运行环境
- `-Xms2G`：初始内存分配2GB
- `-Xmx4G`：最大内存分配4GB
- `-jar server.jar`：运行的核心文件
- `nogui`：不启动图形界面 <Badge type="tip" text="推荐" />

**内存分配建议**：

| 服务器规模 | 最小内存 | 推荐内存 |
|----------|---------|---------|
| 1-5人     | 1G      | 2G      |
| 5-10人    | 2G      | 4G      |
| 10-20人   | 4G      | 6G      |
| 20-50人   | 6G      | 8G+     |
| 50人以上  | 8G      | 16G+    |

### 服务器文件结构

启动后会生成以下文件/文件夹：

::: file-tree
- MyServer
   - server.jar          # 服务器核心
   - eula.txt           # 用户协议
   - server.properties  # 服务器配置文件
   - bukkit.yml         # Bukkit配置（插件服）
   - spigot.yml         # Spigot配置
   - paper.yml          # Paper配置
   - world             # 主世界文件夹
      - region/             # 区域文件（.mca格式）
      - playerdata/         # 玩家数据文件（.dat格式）
      - data/               # 世界数据（如时间、天气等）
      - datapacks/          # 数据包
      - level.dat           # 世界核心信息文件
   - world_nether/      # 下界
   - world_the_end/     # 末地
   - plugins           # 插件文件夹（插件服）
   - mods              # 模组文件夹（模组服）
   - logs              # 日志文件夹
      - latest.log          # 最新日志文件
   - banned-players.json # 封禁玩家列表

:::

## 核心的配置与优化

### 原版配置

#### server.properties

这是服务器的主配置文件，以下是部分重要参数：

```properties title="server.properties"
# 服务器端口(默认25565)
server-port=25565

# 最大玩家数
max-players=20

# 游戏模式(survival/creative/adventure)
gamemode=survival

# 难度(peaceful/easy/normal/hard)
difficulty=normal

# 是否开启正版验证
online-mode=true

# 服务器名称
motd=我的Minecraft服务器

# 视距(区块加载距离)
view-distance=10

# 世界种子
level-seed=

# 是否允许飞行
allow-flight=false

# 是否生成生物
spawn-monsters=true
spawn-animals=true

# 白名单
white-list=false
```

**重要配置说明**：<Badge type="important" text="重要" />

1. **online-mode**: 
   - `true`: 正版验证，只有正版玩家能进入
   - `false`: 离线模式，离线玩家也能进

2. **view-distance**: 
   - 越大越消耗性能
   - 建议: 6-10（普通服务器），4-6（性能较差）

3. **max-players**: 
   - 根据服务器配置设置
   - ==不要设置过高导致卡顿=={.warning}

### 性能优化

#### 1. JVM参数优化

> 详情见：[启动脚本入门](/guide/ji-chu-zhi-shi-xue-xi/startup/)

你可以使用**Aikar's Flags**启动参数作为优化项：

```bash
java -Xms4G -Xmx4G -XX:+UseG1GC -XX:+ParallelRefProcEnabled \
-XX:MaxGCPauseMillis=200 -XX:+UnlockExperimentalVMOptions \
-XX:+DisableExplicitGC -XX:+AlwaysPreTouch \
-XX:G1NewSizePercent=30 -XX:G1MaxNewSizePercent=40 \
-XX:G1HeapRegionSize=8M -XX:G1ReservePercent=20 \
-XX:G1HeapWastePercent=5 -XX:G1MixedGCCountTarget=4 \
-XX:InitiatingHeapOccupancyPercent=15 \
-XX:G1MixedGCLiveThresholdPercent=90 \
-XX:G1RSetUpdatingPauseTimePercent=5 \
-XX:SurvivorRatio=32 -XX:+PerfDisableSharedMem \
-XX:MaxTenuringThreshold=1 -Dusing.aikars.flags=https://mcflags.emc.gs \
-Daikars.new.flags=true -jar server.jar nogui
```

::: note
`Xms` 和 `Xmx` 需要设置为相同值
:::

### 备份策略

#### 自动备份

你可以使用备份插件或脚本定期备份：

**Linux备份脚本示例**：
```bash
#!/bin/bash
DATE=$(date +%Y-%m-%d_%H-%M-%S)
BACKUP_DIR="/backups"
SERVER_DIR="/home/minecraft"

# 创建备份
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz $SERVER_DIR/world

# 删除7天前的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +7 -delete

echo "备份完成: backup_$DATE.tar.gz"
```

**备份频率建议**：
- 世界数据: 每6-12小时
- 插件配置: 每天
- 完整备份: 每周

## 常见问题解答（FAQ）

### 1. 服务器启动失败

**问题描述**：双击启动文件闪退

**解决方案**：
- 检查Java版本是否匹配
- 查看`logs`文件夹中的日志
- 确认内存分配是否合理
- 检查`eula.txt`是否已同意

---

### 2. 无法连接服务器

**问题描述**：玩家无法进入服务器

**解决方案**：
- 检查防火墙是否开放`25565`端口
- 确认服务器IP地址是否正确
- 查看`server.properties`中的端口设置
- 检查路由器端口转发(外网访问)
- 确认游戏版本是否一致

---

### 3. 服务器卡顿

> 详情见：[卡顿入门](/guide/ji-chu-zhi-shi-xue-xi/lag/)

**问题描述**：TPS（每秒刻数）低于20

**解决方案**：
- 使用Spark分析性能瓶颈
- 减少视距（`view-distance`）
- 清理过多实体
- 优化红石机器
- 预生成世界
- 检查插件/模组是否有问题
- 升级服务器配置

---

### 4. 插件冲突

**问题描述**：安装插件后出现报错

**解决方案**：
- 查看日志中的错误信息
- 确认插件版本与核心版本匹配
- 检查是否有功能冲突的插件
- 逐个停用插件排查问题
- 更新插件到最新版本

---

### 5. 内存溢出

**问题描述**：服务器（终端）提示“Out of Memory”

**解决方案**：
- 增加`-Xmx`参数的内存分配
- 减少加载的区块数量
- 清理过期的世界备份
- 优化插件配置
- 考虑升级服务器硬件

---

### 6. 版本更新

**问题描述**：如何升级服务器版本

**步骤**：
1. **完整备份**当前服务器
2. 下载新版本核心
3. 替换旧的核心文件
4. 检查插件兼容性，更新插件
5. 启动服务器测试
6. 如有问题，恢复备份

**注意**：
- 不要跨越太多版本更新
- 某些版本更新会改变世界格式
- 务必备份后再更新

---

### 7. 模组服特有问题

**问题描述**: 模组无法加载

**解决方案**:
- 确认模组版本与Forge/Fabric版本匹配
- 检查模组依赖是否安装
- 查看crash日志
- 确认模组之间无冲突
- 使用整合包管理工具（如CurseForge）
