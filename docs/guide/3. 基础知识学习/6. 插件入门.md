如果你所架设的服务器是用的插件端核心（比如Paper、Spigot、Purpur），则你的服务器可以安装一些插件，以提升原版MC的游戏体验感，以及增添一些玩法。

## 插件是什么东西？
### 简介
顾名思义，插件是==服务器端的一种扩展程序==，用于增加游戏的功能、改善玩家体验或实现特定目的。插件通常由第三方开发者创建，并通过修改服务器的行为来实现其功能。它们==可以添加新的命令、物品、事件等，也可以修改游戏规则、聊天系统等=={.tip}。

插件的作用嘛，我从几个常见的场景讲起吧：

+ **游戏模式**：例如生存模式、创造模式、天空战争、建筑竞赛等。
+ **经济系统**：允许玩家赚取货币、购买物品或与其他玩家交易。
+ **保护机制**：防止破坏、保护玩家建筑或领地。
+ **社交功能**：如投票系统、排行榜、聊天颜色等。
+ **自动化工具**：自动回复、自动备份、自动重启等。
+ **优化性能**：提高服务器性能，减少延迟。

以上就是一些常见场景的插件用途啦~

### 它与模组有什么区别？
插件和模组是两个相对对立的关系户，插件就是插件，模组就是模组，你要知道的是==**它们俩不是一个东西**=={.warning}。

1. **适用环境：**
    - **插件**主要用于服务器端，用来增加服务器的功能，如添加小游戏、聊天管理等。
    - **模组**则安装在玩家的客户端上，用来改变玩家的游戏体验，如增加新的方块、物品或者改变游戏机制。
2. **实现方式：**
    - **插件**通常通过服务器软件（如Bukkit、Spigot、Paper等）提供的API来编写，这些API允许开发者在不修改服务器核心代码的情况下添加新功能。
    - **模组**则是通过修改游戏客户端的Java代码或者使用特定的模组加载器（如Forge, Fabric等）来实现的。
3. **兼容性：**
    - **插件**只需要服务器管理员安装即可，所有连接到该服务器的玩家都能享受到插件带来的功能，而不需要在他们的客户端上做任何更改（除非插件要求客户端安装依赖，比如资源包）。
    - **模组**需要每个玩家在自己的客户端上安装相同的模组才能生效，但是服务端专用的模组，客户端不需要安装。

## 插件分为哪几种？
### 使用场景分类
如果根据使用场景分类，我们可以分为：

1. **Bukkit/Spigot插件**：这是最常见的插件类型，也是一般开发者最常开发的插件类型，是兼容Spigot服务端及其分支服务端（如Paper）的插件；
2. **Paper插件**：专门为Paper服务端设计的插件，比较稀有，且只兼容Paper服务端及其分支服务端（如Purpur）；
3. **BungeeCord插件**：属于群组服插件，只兼容群组服核心（如BungeeCord和WaterFall），不过不兼容Velocity服务端；
4. **Velocity插件**：群组服插件的第二类，就是只兼容Velocity（或基于Velocity分支开发的服务端），不兼容BungeeCord和WaterFall；
5. **Sponge插件**：Sponge是混合端，又称海绵服，属于混合服插件，只兼容Sponge服务端（或基于Sponge分支开发的服务端）；
6. **Nukkit插件**：基岩版插件，只兼容Nukkit服务端；
7. **LiteloaderBDS插件**：基岩版插件，只兼容LiteloaderBDS服务端。

### 标签分类
如果根据标签分类，我们可以分为：

1. **普通插件**：就是插件，普通的插件，可以理解为一支笔壳内的笔芯；
2. **支持库插件**：可以理解为前置插件（笔芯的笔壳）。大部分插件都需要依赖特定的支持库插件来正常运行，就跟中性笔笔芯要依赖笔壳才能正常使用一个道理；
3. **Folia插件**：即支持Folia及其分支服务端的插件，相当于升级版的笔芯。一般而言，它不会与普通插件区分开来，其他服务端仍能正常使用该类插件。

### 功能类别分类
如果根据功能类别分类，我们可以分为：

1. **管理插件**：即管理员专用的服务器管理工具插件，一般被用来运维服务器；
2. **聊天插件**：用来管理聊天的插件，如禁言、脏话屏蔽、聊天前缀，有些还可以为聊天带来新功能和新色彩；
3. **开发者插件**：顾名思义，被插件开发者所利用的插件，一般为支持库插件；
4. **经济插件**：为服务器增添经济功能的插件，如商店、货币、贸易、市场等；
5. **玩法插件**：为服务器带来新玩法的插件，给服务器润色，如技能、动作、图片、新游戏机制等；
6. **小游戏插件**：给服务器带来一些小游戏功能的插件，一般是小游戏服务器的专用插件；
7. **安全插件**：即防熊和服务器保护插件，如反作弊、反压测、反种子破解、反插件查询等；
8. **RPG插件**：即角色扮演插件，一般为RPG服务器专用，生存服务器也可以用；
9. **世界管理插件**：专门管理世界和存档的插件，比如多世界、定时重置、预加载等；
10. **杂项插件**：除上述所有功能的其它功能的插件。

## 如何安装到服务器？
如果你要为你的服务器安装插件，那么你需要遵循以下步骤：

### 下载和获取插件
[📕搜寻需要的插件](https://www.yuque.com/simpfox/mcserverhelp/search-for-plugins)

通过不同渠道下载到插件，这里不多讲解，懂得都懂。

::: danger

不要从一些不正规的渠道获取插件，**里面可能带有后门或其他类型的病毒程序！**

后门的意思是：绕过软件的安全性控制而获取对程序或系统访问权的黑客方法，也就是在未经设备主人的授权下，远程操控设备主人的设备。

:::

### 安装插件
下载好插件后，接下来就是安装插件啦。

一般步骤是：把插件`.jar`文件丢到`plugins`文件夹内，然后重启/启动你的服务器，**查看控制台输出，确认插件是否被正确加载。**

不过你还需要注意：

+ 如果`plugins`文件夹不存在，则需要手动创建。
+ 使用插件自带的命令进行测试，确认一切正常运行。

常见问题：

+ **插件不兼容**：
    - 检查插件是否支持你的服务器版本。
    - 更新服务器软件到最新版本。
+ **插件冲突**：
    - 有些插件可能会和其他插件冲突。
    - 尝试禁用其他插件以确定冲突源。
+ **配置错误**：
    - 配置文件中的错误可能导致插件无法正常工作。
    - 使用文本编辑器仔细检查配置文件语法。

### 配置插件
大多数插件会在==首次启动==时生成配置文件，你可以==编辑这些文件来配置插件==，最常见的配置文件形式是一个名为`config.yml`的文件。

具体点来说就是：插件在服务器启动时，会在`plugins`文件夹内生成跟插件名称相同的文件夹，里面会包含一些配置文件和数据，如果想要配置插件，则需要进入这个文件夹配置。

举个例子：Residence在插件正确加载时，会在`plugins`文件夹内生成一个名为`Residence`的文件夹。

::: tip

一般来说，插件的核心配置文件就是`config.yml`，很多插件都是一样，少部分插件的配置文件的文件名则是诸如`settings.yml`一类的名称。
:::

配置完成后，你的插件就差不多正确安装啦~

## 插件兼容我的服务器吗？
这里就涉及到一个问题了：**插件兼容性**。

想要查看你所需要的插件是否兼容你的服务器，很简单，在插件介绍中就已经可以说明一切了。插件介绍一般会解释插件适配的版本、服务端等等。总而言之就是：**要**==**仔细阅读**==**插件介绍或插件文档，并查找是否有相关的特别说明。**

下面我们分类讨论不同场景：

### SpigotMC上的插件
一般来说，SpigotMC网站上的插件就是专门为`Spigot`服务端设计的，属于`Bukkit/Spigot`插件，可以兼容Spigot服务端及其分支服务端，比如Paper、Purpur、Leaves。

::: warning 不是所有来自SpigotMC网站上的插件都是`Bukkit`/`Spigot`插件

需要注意的一点是，部分插件可能是为Paper或其他服务端专门设计的，只兼容某个服务端及其分支服务端，请仔细阅读插件介绍，并查找是否有相关的特别说明。
:::

### Folia插件
Folia是一种多线程高性能服务端，不过由于是刚起步，所以最初连稳定构建都没有，兼容性较为一般，==大部分插件都不支持Folia服务端=={.info}。

所以你需要：仔细阅读插件介绍，并查找是否有相关的特别说明。

### 论坛帖子插件
论坛的插件也是千奇古怪，什么插件都有，而且提供渠道一般都不稳定，要么是论坛上下载，要么是网盘上下载。反正没有一个统一、有规划的提供渠道就对了。

::: warning

尽量不要在插件论坛下载插件，下载的插件可能包含病毒、后门等危险因素，会爆破你的服务器，能找到官方渠道下载的话就尽量下载官方源插件。
:::

所以你需要：仔细阅读插件介绍，并查找是否有相关的特别说明。

## 如何验证插件兼容性？
这个也很简单，只需要：

1. 把插件丢到`plugins`文件夹
2. 重启服务器
3. 检查终端日志

如果没有出现报错（除了那些没装前置导致的报错、网络问题导致的报错、客观因素导致的报错，比如服务器过载或无法访问Github更新内容）的话，那就是兼容你的服务器。
