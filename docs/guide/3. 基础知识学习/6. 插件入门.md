原版玩腻了？装模组！装插件！为什么？因为这些内容可以提升游戏体验感！

## 1. 基础概念入门

### 1.1 什么是插件和模组?

在开始之前，我们需要明确两个核心概念：

**插件（Plugin）**是==运行在服务器端的扩展程序==，只需要服务器安装，玩家无需安装任何东西就能体验插件功能。插件通常用于服务器管理、经济系统、权限控制等功能。

**模组（Mod）**是对游戏本身的修改，**通常**需要服务器和客户端（玩家）同时安装。模组可以添加新的方块、物品、生物、维度等游戏内容。

### 1.2 插件与模组的主要区别

| 对比项 | 插件 | 模组 |
|--------|------|------|
| 安装位置 | 仅服务器 | 服务器+客户端 |
| 玩家要求 | 无需安装 | 必须安装相同模组 |
| 主要用途 | 管理、经济、保护 | 新内容、新玩法 |
| 兼容性 | 较好 | 需要版本匹配 |
| 学习难度 | 较低 | 较高 |

**简单理解**：插件改变服务器规则，模组改变游戏内容。

### 1.3 适用于哪些服务器核心？

要使用插件或模组，必须先选择合适的服务器核心。这是萌新最容易混淆的地方。

你可以参考这篇教程：[服务器核心入门](/guide/ji-chu-zhi-shi-xue-xi/server-core/)

## 2. 插件系统详解

### 2.1 插件的工作原理

插件通过服务器核心提供的API（应用程序接口）来==修改服务器行为==。当玩家执行某个动作时，插件可以==监听这个事件==并做出反应。

**举例说明**：当一个玩家尝试破坏某个受到领地保护插件所保护的一个方块时，领地保护插件会检查该玩家是否有权限，如果没有权限就取消这个动作。

### 2.2 插件文件结构

一个标准的插件通常是 `.jar` 文件，内部包含：
- 编译好的Java代码
- `plugin.yml` 配置文件（插件信息）
- 资源文件（语言文件、默认配置等）

### 2.3 如何安装插件

::: steps
1. 确保你使用的是支持插件的服务器核心（如Paper、Spigot）

2. 下载插件的 `.jar` 文件
   - 官方来源：Spigot官网、Bukkit官网
   - 第三方来源：GitHub、CurseForge等

3. 将 `.jar` 文件放入服务器的 `plugins` 文件夹

4. 重启服务器或使用热加载命令
   ```
   /reload 或 /restart
   ```

5. 检查插件是否加载成功
   ```
   /plugins
   ```
:::

### 2.4 插件配置文件

大多数插件安装后会在 `plugins/插件名称/` 文件夹下生成配置文件，通常是一个名为 `config.yml` 的YAML格式的文件。

**配置文件示例**（以权限插件为例）:
```yaml
# 是否启用调试模式
debug: false

# 默认用户组
default-group: member

# 用户组配置
groups:
  admin:
    permissions:
      - "*"
  member:
    permissions:
      - "essentials.home"
      - "essentials.spawn"
```

**编辑注意事项**:
- 使用支持YAML的文本编辑器（推荐Notepad++、VS Code）
- 注意缩进，必须使用空格，不能使用Tab键
- 修改后需要重载插件或重启服务器

### 2.6 常见问题（FAQ）

::: steps

1. **问题1**：插件名显示红色，加载失败
   - **原因**：可能是缺少前置插件或版本不兼容？
   - **解决**：查看控制台错误信息，安装所需前置

2. **问题2**：插件命令无法使用
   - **原因**：可能是权限配置错误？
   - **解决**：给予自己管理员权限 `/lp user 你的名字 permission set * true`

3. **问题3**：修改配置文件后不生效
   - **原因**：可能是未重载或YAML格式错误？
   - **解决**：使用 `/插件名 reload` 或检查YAML语法

:::

## 3. 模组系统详解

### 3.1 模组的工作原理

模组（Mod）直接修改游戏代码或添加新代码，可以==实现插件无法完成的功能==，如新方块、新生物、新维度等。

### 3.2 模组加载器详解

#### 3.2.1 Forge
- **特点**：历史最悠久，模组数量最多
- **版本**：每个MC版本对应不同的Forge版本
- **下载**：[files.minecraftforge.net](https://files.minecraftforge.net/)
- **安装**：
  1. 下载对应版本的Forge安装器
  2. 选择"Install Server"
  3. 指定服务器文件夹
  4. 运行生成的启动脚本

#### 3.2.2 Fabric
- **特点**：轻量级，更新迅速，适合新版本
- **前置模组**：需要安装Fabric API
- **下载**：fabricmc.net
- **优势**：加载速度快，内存占用低

#### 3.2.3 版本选择建议
- 1.12.2 → Forge（模组最多）
- 1.16.5 → Forge（成熟稳定）
- 1.18+ → Fabric或Forge均可
- 1.20+ → 推荐Fabric（更新更快）

### 3.3 如何安装模组

**客户端模组安装**:
1. 安装对应的Forge/Fabric客户端
2. 将模组 `.jar` 文件放入 `.minecraft/mods` 文件夹
3. 启动游戏

**服务器模组安装**:
1. 安装Forge/Fabric服务器核心
2. 将模组`.jar`文件放入服务器`mods`文件夹
3. 启动服务器

::: warning
玩家客户端必须安装相同的模组才能进入（除去双端模组之外）
:::

### 3.5 必备前置模组

许多模组不能单独安装，需要前置模组才能正常运行：

#### Forge系
- **Forge本身**：基础加载器
- **Kotlin for Forge**：部分模组需要
- **Architectury API**：跨平台模组前置

#### Fabric系
- **Fabric API**：几乎所有Fabric模组都需要
- **Fabric Language Kotlin**：Kotlin编写的模组需要
- **Cloth Config API**：配置界面前置

### 3.7 常见问题（FAQ）

::: steps

1. **问题1**：玩家无法进入服务器
   - **原因**：模组不一致
   - **解决**：确保客户端和服务器模组完全相同（版本号也要一致）

2. **问题2**：服务器启动崩溃
   - **原因**：模组冲突或缺少前置
   - **解决**：查看崩溃报告（`crash-reports`文件夹），移除冲突模组

3. **问题3**：内存不足
   - **原因**：模组服务器需要更多内存
   - **解决**：增加启动参数中的内存分配
   ```bash
   java -Xmx4G -Xms4G -jar forge-server.jar
   ```
   （`4G`不是一个固定值，你可根据实际需求而调整为`6G`、`8G`等）

:::

## 4. 混合端（插件+模组）

### 4.1 为什么需要混合端？

插件提供管理功能，模组提供游戏内容，两者结合可以打造==更完善的服务器体验==。也就是互补关系，互相补充各自不足的地方。

### 4.2 混合核心选择

**Mohist**（1.7.10，1.12.2，1.16.5-1.20.1）：
- 支持Forge模组 + Bukkit插件
- 稳定性一般，部分模组或插件可能不兼容

**CatServer**（1.12.2，1.16.5，1.18.2）：
- 支持Forge模组 + Spigot插件
- 中文社区支持较好

**Arclight**（1.14 - 1.20）：
- 现代化混合核心
- 兼容性相对较好

### 4.3 兼容性问题

:::: card-grid

::: card title="需要注意的是"
   - 领地保护插件可能无法保护模组方块
   - 部分模组的自定义维度与插件冲突
   - 模组物品可能无法在插件商店中正常交易
:::

::: card title="而你需要"
   1. 优先选择官方声明兼容的插件
   2. 使用模组自带的保护系统替代插件
   3. 测试每个插件和模组的兼容性
   4. 查看混合核心官方的兼容性列表
:::

::::

## 5. 故障排查

### 5.1 插件加载失败

**症状**：插件显示红色，插件功能也不可用

**排查步骤**：
1. 查看控制台错误信息
2. 检查是否缺少前置插件
3. 确认服务器核心版本兼容性
4. 尝试更新到最新版本插件

**示例错误**：
```
Could not load 'plugins/XXX.jar'
→ 插件损坏或版本不兼容

java.lang.NoClassDefFoundError: Vault
→ 缺少Vault前置插件
```

### 5.2 模组冲突

**症状**：服务器崩溃，crash report生成

**解决流程**：
1. 打开 `crash-reports` 文件夹
2. 查看最新的崩溃报告
3. 找到 "A detailed walkthrough" 部分
4. 查看堆栈跟踪,找到冲突的模组名称
5. 移除或更新冲突模组

**二分法排查**：
- 移除一半模组测试
- 确定哪一半有问题
- 继续二分直到找到具体模组

## 6. 进一步讨论 <Badge type="tip" text="进阶" />

### 6.1 多世界管理

想要给你的服务器添加一个多世界功能？只需要安装一个多世界插件就行了！

**例如**：Multiverse-Core、MyWorlds等插件就包含了多世界功能

**用途**：
- 创建多个独立世界
- 不同世界不同规则（如主世界当家园世界用来建家，而资源世界开放破坏）

**基础命令**：
```
/mv create 世界名 normal - 创建主世界
/mv create 资源世界 normal - 创建资源世界
/mv tp 世界名 - 传送到世界
```

### 6.2 数据库集成

数据库（Database）是结构化信息或数据的有序集合，通常以电子形式存储于计算机系统中。它使用户能够高效地存储、检索、管理和操作数据。

**例如**：Plan，LuckPerms等插件就支持MySQL数据库

**为什么需要数据库**？
- 多服数据共享
- 数据持久化
- 高性能查询

**配置示例** (LuckPerms)：
```yaml
storage-method: mysql
data:
  address: localhost:3306
  database: minecraft
  username: root
  password: yourpassword
```

### 6.3 自定义资源包

**你可以在服务器配置中完成操作**：
```properties
# server.properties
resource-pack=https://你的资源包URL.zip
resource-pack-sha1=资源包SHA1值
require-resource-pack=false
```

### 6.4 学习开发插件？

如果你有开发插件这一方面的兴趣爱好，或是想制作一个自己的插件，那么你可以试着学习一下下……

**那么你需要准备**:
- IntelliJ IDEA 或 Eclipse
- Maven / Gradle
- Java开发基础

**创建第一个插件**:
```java
public class MyFirstPlugin extends JavaPlugin {
    @Override
    public void onEnable() {
        getLogger().info("插件已启动!");
    }
    
    @Override
    public void onDisable() {
        getLogger().info("插件已关闭!");
    }
}
```

## 7. 一个简单的总结

**❌ 常见错误**：
1. 安装过多插件导致冲突
2. 随意给予OP权限
3. 不做备份就进行重大更改
4. 使用盗版或来源不明的插件
5. 服务器配置与机器性能不匹配

**✅ 正确做法**：
1. 循序渐进，一个个测试插件
2. 使用权限系统精细化管理
3. 定期备份，修改前先备份
4. 只从官方渠道下载
5. 根据玩家数量和硬件合理配置
