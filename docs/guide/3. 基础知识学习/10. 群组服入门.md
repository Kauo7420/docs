## 什么是群组服？
群组服本质上是一个由多个子服务器组成的网络，每个子服务器可以拥有不同的游戏模式、规则和世界。通过中央代理软件，如BungeeCord或Waterfall，玩家可以在这些子服务器之间自由切换，享受从生存模式到迷你游戏，再到PvP竞技场等多样的游戏体验，无需重新连接或登录。

### 群组服的优势
+ **多样性**：提供多种游戏模式，满足不同玩家的偏好。
+ **社区感**：统一的登录和权限系统增强了玩家之间的连通性和归属感。
+ **资源效率**：通过负载均衡和资源共享，优化服务器性能和成本。
+ **扩展性**：易于添加新的子服务器，引入更多的游戏内容和特色。

### 中央代理软件
+ **BungeeCord**：最初由SpigotMC团队开发，是最早和最流行的Minecraft群组服代理软件之一。
+ **Velocity**：由PaperMC团队创建，旨在提供比BungeeCord更好的性能和更现代的特性支持。

### 玩家体验
+ **无缝切换**：玩家可以从一个子服务器轻松跳转至另一个，享受流畅的游戏体验。
+ **统一账户**：无论在哪一个子服务器上，玩家的账号信息、权限和物品都是通用的。
+ **社交互动**：跨服务器聊天、交易和竞赛促进了玩家之间的交流和合作。

## 群组服和单端服的区别？
在Minecraft服务器生态中，群组服和单端服代表了两种截然不同的游戏体验和管理模式。每种服务器类型都有其独特的优势和适用场景，了解它们之间的区别有助于服主和玩家做出更适合自己的选择。

### 单端服（Single Server）
单端服是最基本的Minecraft服务器类型，它提供了一个独立的游戏环境，通常专注于一种特定的游戏模式，如生存（Survival）、创造（Creative）、冒险（Adventure）或PvP（Player vs Player）。

+ **特点**：
    - **集中式体验**：玩家在一个统一的世界里进行游戏，通常具有固定的规则和目标。
    - **简单管理**：由于只有一台服务器，资源管理、维护和升级相对直接。
    - **社区专注**：玩家群体围绕特定的游戏模式形成紧密的社区。
+ **优势**：
    - 对于特定游戏模式的深入体验。
    - 资源消耗较低，适合小规模玩家群体。
    - 更加专注的社区和玩家互动。

### 群组服（Network Server）
群组服则是一种高级服务器架构，它由多个独立的子服务器组成，每个子服务器可以运行不同的游戏模式或世界。玩家可以通过中央代理在这些子服务器之间自由切换，享受多样化的游戏体验。

+ **特点**：
    - **多样性**：提供多种游戏模式，满足不同玩家的偏好，如生存、创造、PvP、小游戏等。
    - **资源优化**：通过负载均衡和资源调度，提高服务器效率和玩家体验。
    - **扩展性**：易于添加新的子服务器，引入更多游戏内容和特色。
+ **优势**：
    - 提供广泛的娱乐选项，适合大型玩家社区。
    - 高效的资源管理和负载均衡。
    - 促进玩家之间的社交互动和跨游戏模式的竞赛。

### 主要区别
+ **游戏模式**：单端服专注于一种游戏模式，而群组服提供多种游戏模式。
+ **资源管理**：单端服的资源管理较为简单，群组服则需要更复杂的资源配置和优化。
+ **社区结构**：单端服的社区围绕单一游戏模式构建，群组服的社区更加多元化，涵盖各种游戏爱好者。
+ **技术难度**：单端服的搭建和维护相对容易，群组服则需要更高级的技术知识和服务器管理技巧。

::: tip 抽象化理解

单端服就是一个独立的服务器，比如Paper、Spigot、Vanilla等服务端都是单端服务器；

群组服就是很多个独立的服务器结合在一起，组成一个群组，因此得名“群组服”，比如BungeeCord和Velocity就是群组服务端。

你可以理解为群组服把很多个单端服结合成了一个群体，形成了一个整体，这就是群组服服务器的本质。

:::

## 如何架设群组服？
### 架设前的准备
+ **硬件评估**：确保你的服务器硬件（CPU、RAM、硬盘空间）足以支撑预期的玩家数量和游戏模式。
+ **软件选择**：选择合适的群组服代理软件，如BungeeCord或Velocity。
+ **规划网络结构**：决定你想要提供的游戏模式和服务，规划子服务器的数量和类型。

### Step 1. 安装代理软件
1. **下载代理软件**：访问官方网站获取BungeeCord或Velocity的最新版本。
2. **配置代理**：编辑`config.yml`文件，设置监听端口、插件目录、子服务器列表等参数。

### Step 2. 设置子服务器
1. **创建子服务器文件夹**：为每种游戏模式创建独立的文件夹，如“Survival”、“Creative”、“MiniGames”。
2. **配置子服务器**：为每个子服务器设置`server.properties`，包括世界生成、游戏规则等。
3. **安装必要插件**：根据游戏模式需求安装相应的插件，如WorldEdit、Essentials等。

### Step 3. 连接子服务器
1. **添加子服务器信息**：在代理的`proxyhosts.yml`文件中，添加每个子服务器的名称、IP地址和端口。
2. **设置跳转规则**：定义玩家登录后默认进入的服务器，以及如何在玩家请求时切换服务器。

### Step 4. 启动与测试
1. **启动所有服务**：先启动子服务器，然后启动代理软件。
2. **测试连接**：使用Minecraft客户端连接，确保所有子服务器都能被正确识别和访问。

## 如何维护群组服？
+ **定期更新**：保持代理软件、插件和游戏核心的最新版本，以获得最新的功能和安全性。
+ **数据库管理**：如果使用数据库存储玩家数据，定期进行备份和优化，确保数据安全和访问速度。
+ **日志分析**：当遇到问题时，检查代理和子服务器的日志文件，寻找错误信息。

## 如何配置群组服的数据库？
在MC服中，数据库用于存储玩家信息、游戏进度、经济系统、排行榜等各种数据。正确配置和调整数据库是确保服务器数据安全和性能的关键。本教程将指导你如何选择、配置和优化数据库系统。

### 选择数据库类型
+ **SQLite**：轻量级的文件型数据库，适合小型服务器或不需要高并发的环境。
+ **MySQL/MariaDB**：适用于中大型服务器，支持多用户访问，具有更好的性能和扩展性。
+ **PostgreSQL**：开源的关系型数据库，适用于需要复杂查询和高安全性的场景。

### 数据库配置步骤
#### SQLite
1. **创建数据库文件**：通常，插件会自动创建一个`.db`或`.sqlite`文件作为数据库。
2. **配置插件**：编辑插件的配置文件，指定数据库文件的路径。

#### MySQL/MariaDB 或 PostgreSQL
1. **安装数据库服务**：在服务器上安装并配置MySQL/MariaDB或PostgreSQL服务。
2. **创建数据库**：使用数据库管理工具或命令行创建一个新的数据库。
3. **创建用户和权限**：为Minecraft服务器创建专用的数据库用户，并赋予其对新创建数据库的读写权限。
4. **配置插件**：在插件的配置文件中填写数据库的连接信息，包括主机名、端口、数据库名、用户名和密码。

### 性能优化
+ **索引**：为经常查询的字段创建索引，可以显著提高查询速度。
+ **缓存**：利用缓存机制减少频繁的数据库读取操作，例如使用Redis或Memcached。
+ **优化查询**：避免全表扫描，使用JOIN语句时要谨慎，尽量减少嵌套查询。

### 安全性
+ **限制权限**：只给数据库用户授予必要的最小权限，避免使用root账号。
+ **加密通信**：使用SSL/TLS加密数据库连接，保护数据传输过程中的安全。
+ **防火墙规则**：只允许特定的IP地址访问数据库，最好限制为仅服务器本身。

### 备份与恢复
+ **定期备份**：设定自动备份策略，至少每天备份一次数据库，以防数据丢失。
+ **测试恢复流程**：定期从备份中恢复数据，验证备份的有效性。

### 监控与日志
+ **性能监控**：使用数据库管理工具或第三方监控服务，监控数据库的性能指标。
+ **日志记录**：开启数据库的日志记录，以便于问题排查和审计。
