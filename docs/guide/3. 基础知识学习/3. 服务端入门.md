选择正确的Minecraft服务器核心（服务端）是构建成功服务器的第一步。不同的服务端软件各有特色，适用于不同场景和需求。本指南将帮助你了解并选择最适合你的服务器核心，帮你踏出正确的第一步，拥有一个良好开端。

## 什么是服务器核心？

### 基础概念

**服务器核心**就是==让Minecraft能够以多人游戏模式运行的程序文件==。如果把Minecraft服务器比作一栋房子, 那么服务器核心就是这栋房子的地基和框架。

简单来说:
- 你平时玩的Minecraft客户端 = 单人游戏
- 服务器核心 = 让多人可以同时进入同一个世界游戏的程序

### 核心的本质

服务器核心通常是一个 `.jar` 文件(Java程序),它包含了:
- 游戏世界的运算逻辑
- 玩家连接管理
- 方块、生物、物品等游戏元素的处理
- 指令系统
- 插件/模组接口(部分核心)

## 服务器核心的作用

### 核心功能

::: note 防歧义
需要注意的是，这里的核心指的是服务器核心，而不是汉语词语中指代中心或主要部分的“核心”。
:::
1. **世界管理**: 生成地形、保存数据、处理区块加载
2. **玩家管理**: 处理玩家登录、移动、互动
3. **游戏逻辑**: 计算伤害、生物AI、红石电路等
4. **网络通信**: 接收客户端数据、发送世界更新
5. **扩展支持**: 提供插件或模组的运行环境(部分核心)

### 为什么需要选择核心？

Mojang官方提供的服务器核心(Vanilla)功能有限, 因此社区开发了各种增强版核心, 它们提供:
- 更好的性能优化
- 插件支持(添加各种功能)
- 模组支持(改变游戏玩法)
- 反作弊功能
- 更多管理工具

## 主流服务器核心类型

### 1. 官方原版核心

**代表**: Vanilla (官方服务端)

**特点**:
- Mojang官方发布
- 100%原版游戏体验
- 不支持插件和模组
- 性能一般

**适用场景**:
- 纯原版生存服务器
- 小型私人服务器(朋友联机)
- 不需要任何扩展功能

**优点**: 最稳定、最纯粹
**缺点**: 功能少、性能差、无法防作弊

---

### 2. Bukkit系核心(插件服)

这是最流行的插件服核心体系,经过多年发展已经非常成熟。

#### 2.1 CraftBukkit

**特点**:
- 最早的插件服核心
- 支持Bukkit插件
- 现在基本被淘汰

**现状**: 已经很少使用,被后继者取代

---

#### 2.2 Spigot

**特点**:
- CraftBukkit的优化版
- 支持Bukkit和Spigot插件
- 有一定的性能优化
- 社区活跃, 更新及时

**适用场景**:
- 中小型插件服务器
- 需要基础插件功能
- 配置要求不高的服务器

**优点**:
- 插件生态成熟
- 稳定性好
- 文档丰富

**缺点**:
- 性能优化不如Paper
- 大型服务器可能卡顿

**官网**: [https://www.spigotmc.org/](https://www.spigotmc.org/)

---

#### 2.3 Paper (推荐)

**特点**:
- Spigot的高性能分支
- 目前最流行的插件服核心
- 大量性能优化和bug修复
- 完全兼容Spigot/Bukkit插件
- 提供额外的API

**适用场景**:
- 中大型插件服务器
- 需要高性能的服务器
- 对稳定性有要求

**优点**:
- 性能优秀
- 开发团队专业
- 更新快速
- 修复了很多原版bug

**缺点**:
- 部分插件可能不兼容(极少数)
- 某些机制与原版略有不同

**官网**: [https://papermc.io/](https://papermc.io/)

---

#### 2.4 Purpur

**特点**:
- Paper的分支
- 提供更多游戏性配置选项
- 性能优化更激进
- 可自定义的功能更多

**适用场景**:
- 需要高度自定义的服务器
- 追求极致性能

**优点**:
- 配置选项极其丰富
- 性能优于Paper

**缺点**:
- 某些优化可能影响原版特性
- 相对小众

**官网**: [https://purpurmc.org/](https://purpurmc.org/)

---

### 3. 模组服核心

模组服核心允许加载Mod, 改变游戏玩法。

#### 3.1 Forge

**特点**:
- 最经典的模组加载器
- 支持绝大多数Mod
- 生态系统最庞大

**适用场景**:
- 需要运行大型模组包
- 科技、魔法、冒险类模组服

**优点**:
- 模组数量最多
- 文档和教程丰富

**缺点**:
- 性能较差
- 启动速度慢

**官网**: [https://files.minecraftforge.net/](https://files.minecraftforge.net/)

---

#### 3.2 Fabric

**特点**:
- 新一代轻量级模组加载器
- 启动快、性能好
- 模组生态快速发展

**适用场景**:
- 轻量级模组服
- 需要快速更新到新版本
- 注重性能的模组服

**优点**:
- 性能优秀
- 更新速度快
- 代码现代化

**缺点**:
- 模组数量不如Forge
- 部分老模组不支持

**官网**: [https://fabricmc.net/](https://fabricmc.net/)

---

#### 3.3 NeoForge

**特点**:
- Forge的现代化分支
- 改进了Forge的很多问题
- 逐渐获得模组开发者支持

**适用场景**:
- 新版本模组服
- 追求更好性能的Forge用户

**现状**: 正在快速发展中

**官网**: [https://neoforged.net/](https://neoforged.net/)

---

### 4. 混合核心(插件+模组)

这类核心同时支持插件和模组, 但==兼容性可能存在问题=={.warning}。

#### 4.1 Mohist

**特点**:
- Forge + Paper的混合核心
- 同时支持Mod和插件

**适用场景**:
- 既需要模组又需要插件功能

**优点**:
- 功能全面

**缺点**:
- 兼容性问题多
- 可能不稳定
- Bug较多

**官网**: [https://mohistmc.cn/](https://mohistmc.cn/)

::: warning
不推荐新手使用, 问题排查困难
:::

---

#### 4.2 CatServer

**特点**:
- 国人开发
- Forge + Spigot混合

**现状**: 已停止维护

**官网**: [https://catmc.org/](https://catmc.org/)

---

#### 4.3 Arclight

**特点**:
- Forge/Fabric + Bukkit混合
- 支持多个版本

**适用场景**:
- 模组+插件混合服

**官网**: [https://arclight.izzel.io/](https://arclight.izzel.io/)

::: warning
兼容性问题需要仔细测试
:::

---

### 5. 基岩版核心

用于Minecraft基岩版(手机版、Win10版)的服务器。

#### 5.1 BDS (官方)

**特点**:
- 官方基岩版服务端
- 不支持插件

#### 5.2 Nukkit系列

**特点**:
- 基岩版的插件服核心
- 支持基岩版插件

## 如何选择合适的核心

::: note
以下内容仅供参考, 请以实际情况为准
:::

### 决策流程图

```
想开什么类型的服务器?
│
├─ 纯原版生存 → Vanilla 或 Paper(优化更好, 但部分特性机制与原版不同)
│
├─ 插件服(小游戏、RPG、生存辅助等)
│   │
│   ├─ 小型服务器(10人以下) → Spigot
│   │
│   └─ 中大型服务器 → Paper(推荐) 或 Purpur
│
├─ 模组服(科技、魔法、冒险等)
│   │
│   ├─ 大型模组包 → Forge
│   │
│   └─ 轻量模组/新版本 → Fabric
│
└─ 插件+模组混合 → Mohist/Arclight(谨慎使用)
```

### 选择建议

**新手服主优先选择顺序**:

1. **Paper** - 适合90%的插件服需求
2. **Fabric** - 轻量模组服首选
3. **Forge** - 大型模组包必选
4. **Purpur** - 进阶用户, 需要更多自定义
5. **Spigot** - Paper的备选方案

**避免选择**:
- 混合核心(除非你很清楚自己在做什么)
- 已停更的核心
- 来源不明的核心

## 核心的下载与安装

### 准备工作

#### 1. 确认游戏版本

首先确定你要开的服务器版本(如1.20.1), 核心版本必须与客户端版本一致。

#### 2. 安装Java

不同Minecraft版本需要不同的Java版本:

| Minecraft版本 | 所需(最低)Java版本 |
|--------------|-------------|
| 1.12-1.16.5 | Java 8  |
| 1.17-1.17.1 | Java 16 |
| 1.18-1.20.4 | Java 17 |
| 1.20.5-1.21.11 | Java 21 |
| 26.1+ | Java 25 |

**Java下载**:
- 推荐使用 Adoptium (OpenJDK): [https://adoptium.net/](https://adoptium.net/)
- 下载对应版本的JDK(Java Development Kit)

**检查Java版本**:
```bash
java -version
```

---

### 下载核心

#### Paper下载示例

1. 访问官网: [https://papermc.io/downloads](https://papermc.io/downloads)
2. 选择游戏版本(如1.20.1)
3. 点击下载最新构建版本
4. 得到文件如: `paper-1.20.1-196.jar`

#### Fabric下载示例

1. 访问官网: [https://fabricmc.net/use/server/](https://fabricmc.net/use/server/)
2. 选择游戏版本
3. 下载Fabric服务端安装器
4. 运行安装器, 选择安装路径
5. 生成服务端文件

#### Forge下载示例

1. 访问: [https://files.minecraftforge.net/](https://files.minecraftforge.net/)
2. 选择Minecraft版本
3. 下载Installer(安装器)
4. 运行安装器:
   ```bash
   java -jar forge-installer.jar --installServer
   ```
5. 等待安装完成

---

### 安装步骤

#### 方法一: 直接运行(Paper/Spigot等)

1. **创建服务器文件夹**:
   ```
   新建文件夹,如: MyServer
   ```

2. **放入核心文件**:
   ```
   将下载的 .jar 文件放入文件夹
   重命名为 server.jar (可选,便于管理)
   ```

3. **首次启动**:
   
   Windows系统创建 `start.bat`:
   ```batch
   @echo off
   java -Xms2G -Xmx4G -jar server.jar nogui
   pause
   ```
   
   Linux系统创建 `start.sh`:
   ```bash
   #!/bin/bash
   java -Xms2G -Xmx4G -jar server.jar nogui
   ```

4. **同意EULA**:
   
   首次运行会生成 `eula.txt`, 打开文件:
   ```
   将 eula=false 改为 eula=true
   ```

5. **再次启动**:
   
   运行启动脚本, 服务器开始生成世界

---

#### 参数说明

```bash
java -Xms2G -Xmx4G -jar server.jar nogui
```

- `java`: 调用Java运行环境
- `-Xms2G`: 初始内存分配2GB
- `-Xmx4G`: 最大内存分配4GB
- `-jar server.jar`: 运行的核心文件
- `nogui`: 不启动图形界面(推荐)

**内存分配建议**:

| 服务器规模 | 最小内存 | 推荐内存 |
|----------|---------|---------|
| 1-5人     | 1G      | 2G      |
| 5-10人    | 2G      | 4G      |
| 10-20人   | 4G      | 6G      |
| 20-50人   | 6G      | 8G+     |
| 50人以上  | 8G      | 16G+    |

---

### 服务器文件结构

启动后会生成以下文件/文件夹:

::: file-tree

    - MyServer
        - server.jar          # 服务器核心
        - eula.txt           # 用户协议
        - server.properties  # 服务器配置文件
        - bukkit.yml         # Bukkit配置(插件服)
        - spigot.yml         # Spigot配置
        - paper.yml          # Paper配置
        - world/             # 主世界文件夹
        - world_nether/      # 下界
        - world_the_end/     # 末地
        - plugins/           # 插件文件夹(插件服)
        - mods/              # 模组文件夹(模组服)
        - logs/              # 日志文件夹
        - banned-players.json # 封禁玩家列表

:::

## 常见问题解答

### 1. 服务器启动失败

**问题**: 双击启动文件闪退

**解决方案**:
- 检查Java版本是否匹配
- 查看`logs`文件夹中的日志
- 确认内存分配是否合理
- 检查`eula.txt`是否已同意

---

### 2. 无法连接服务器

**问题**: 玩家无法进入服务器

**解决方案**:
- 检查防火墙是否开放`25565`端口
- 确认服务器IP地址是否正确
- 查看`server.properties`中的端口设置
- 检查路由器端口转发(外网访问)
- 确认游戏版本是否一致

---

### 3. 服务器卡顿

**问题**: TPS(每秒刻数)低于20

**解决方案**:
- 使用Spark分析性能瓶颈
- 减少视距(`view-distance`)
- 清理过多实体
- 优化红石机器
- 预生成世界
- 检查插件/模组是否有问题
- 升级服务器配置

---

### 4. 插件冲突

**问题**: 安装插件后出现报错

**解决方案**:
- 查看日志中的错误信息
- 确认插件版本与核心版本匹配
- 检查是否有功能冲突的插件
- 逐个停用插件排查问题
- 更新插件到最新版本

---

### 5. 内存溢出

**问题**: 服务器提示"Out of Memory"

**解决方案**:
- 增加`-Xmx`参数的内存分配
- 减少加载的区块数量
- 清理过期的世界备份
- 优化插件配置
- 考虑升级服务器硬件

---

### 6. 版本更新

**问题**: 如何升级服务器版本

**步骤**:
1. **完整备份**当前服务器
2. 下载新版本核心
3. 替换旧的核心文件
4. 检查插件兼容性,更新插件
5. 启动服务器测试
6. 如有问题,恢复备份

**注意**: 
- 不要跨越太多版本更新
- 某些版本更新会改变世界格式
- 务必备份后再更新

---

### 7. 模组服特有问题

**问题**: 模组无法加载

**解决方案**:
- 确认模组版本与Forge/Fabric版本匹配
- 检查模组依赖是否安装
- 查看`crash`日志
- 确认模组之间无冲突
- 使用模组包管理工具(如CurseForge)

## 总结

*服务器核心只是你入门的一个开始, 但开服的道路还很长……*

1. **从小做起**: 先开小型服务器积累经验
2. **多做备份**: 备份永远不嫌多
3. **持续学习**: 关注社区动态和更新
4. **寻求帮助**: 遇到问题不要害怕提问
5. **享受过程**: 开服是一个学习和成长的过程
