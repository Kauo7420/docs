选择正确的Minecraft服务器核心（服务端）是构建成功服务器的第一步。不同的服务端软件各有特色，适用于不同场景和需求。本指南将帮助你了解并选择最适合你的服务器核心，帮你踏出正确的第一步，拥有一个良好开端。

## 服务端是什么？
服务端，即服务器核心（又称服务器软件），是服务器运行的核心部分，相当于一台电脑的CPU，没有它，服务器就会群龙无首，无法运行。

总而言之呢，意思就是服务器全靠一个服务器核心运行，没有服务器核心就没有服务器。

按照服务器功能来划分服务端核心，可分为：

1. **官方端**：由Mojang官方开发并发布的服务器核心，纯净无添加，但是不支持插件和模组，追求原汁原味玩法的人可以一试。
2. **插件端**：由第三方开发者开发并发布的服务器核心，支持在服务器上安装插件，但是不支持模组，是大部分服主的第一选择。
3. **模组端**：由模组加载器开发者开发并发布的服务器核心，支持在服务器上安装模组，但是不支持插件，是模组服服主的绝佳选择。
4. **混合端**：由第三方开发者开发并发布的服务器核心，支持在服务器上同时安装插件和模组，但是稳定性和兼容性较差，是部分追求个性化的服主的选择。
5. **代理端**：由第三方开发者开发并发布的服务器核心，这个服务端很特殊，不能单独作为一个服务器运行，他是联通不同服务器之间的一个代理，也就是部分人口中所说的“群组服”。

::: info

除此之外，你还需要知道的是，混合端一般都是有自身的专属插件，如Sponge插件等。

:::

::: tip 分支是什么意思？

以MC服务端层面为基准，我们假设说A服务端是B服务端的分支，那么这句话的意思就是A服务端是基于B服务端开发的，基于B服务端二创（二次创作）而来，比如在原有的功能上添加一些额外的功能，或删去一些原有的功能，不过整体来说，分支服务端并不会过多改变原服务端的整体面貌。

:::

### 服务端讲解
如果你真正想要了解一个服务端，那么你必须去实践试用一段时间后你才能真正得知这个服务端的利弊和效果。

所以我在这里列出了一些服务端黑话，小白服主可以参考一下：

+ ~~**CraftBukkit**~~：如果是最近才入坑服务器的服主的话，那么我估计你没听说过这个服务端，这个服务端由Bukkit团队开发，集成了Bukkit API，开创了插件服的时代，是所有插件服务端的老祖，不过Bukkit项目后来被Mojang Studios收购，并在一段时间内得到了官方的支持，_目前网上提供的CraftBukkit下载入口真实性不明，据说可能因为反编译（是违反Minecraft EULA的行为）而EOL了_。
+ **Spigot**：插件兼容王，由SpigotMC团队和Bukkit团队开发，大部分插件都能兼容，不过不怎么推荐使用；
+ **Paper**：单线程性能王，在性能方面表现的很好，很适合小服，不过破坏了很多原版特性，不适合生电服，如果是大一点的服务器就请移步至Folia；
+ **Folia**：多线程性能王，新兴服务端，在多线程性能方面表现得很好，很适合大服，不过破坏的原版特性比Paper更多，比如禁用了数据包和fuction；
+ **Purpur**：Paper的完美替代品，可配置性更强，适合追求原版特性的生存玩法和小型生电玩法，不过大型生电玩法建议移步Leaves服务端；
+ **Leaves**：生电王，大量修复了Paper破坏的原版特性，并且还往里面塞了一个钠（Lithium），是追求生电服的良好选择，不过不是生电服的第一选择（Fabric才是）。
+ **Folia**：新兴的服务端，由PaperMC团队开发，截至目前还没有一个官方稳定构建，特色功能是解决了MC服的单线程性能瓶颈问题，旨在强化多线程性能，让服务器性能飞一样的提升，不过破坏的原版特性较多，比如禁用了`bossbar`，`clone`，`data`，`debug`，`item`，`loot`，`ride`，`reload`，`schedule`，`scoreboard`，`tag`，`team`等，最烦人的是禁用了数据包和`fuction`；
+ **Lumina**：由LeavesMC团队开发，Folia的分支服务端，旨在修复Folia所破坏的原版特性，适配生电，相当于Leaves服务端的多线程优化版本。但它同样附带Folia的的特点，也就是说，Folia有什么缺点，Lumina就有什么缺点。比如，Lumina跟Folia一样，兼容性较差。

## 如何挑选服务端？
如果要挑选一个适合你服务器的服务端，那么你就需要根据上面思维导图所列出的服务器核心，结合你的服务器的实际情况，选择正确的服务器核心。下面给出了一些推荐的组合：

### 按服务器玩法类型挑选
#### A. 生存服
如果你的服务器是生存服，那么我推荐：

1. **Vanilla**: 纯净生存专用，Mojang官方提供的服务器核心，不过不支持插件*（悲）*。
2. **Spigot**: 插件服生存适用，插件兼容性好，且性能高。
3. **Leaves**: 推荐追求原汁原味玩法的插件服服主使用，它支持原版特性。
4. **Paper**: 追求极致性能的话可以一试，这个端有着高度优化。

#### B. 模组服
如果你的服务器是模组服，那么我推荐：

1. **Forge**、**Fabric**、**NeoForge**: 都是模组加载器，懂得都懂，视情况而定。
2. **Sponge**: 想要插件和模组共存的话，可以一试，不过不能保证所有插件和模组的兼容性。

#### C. 生电服
如果你的服务器需要建造大量的红石机械（或是以红石机械为主要玩法），那么我推荐：

1. **Fabric**: 懂得都懂，Fabric绝对是生电服的完美服务端，它是生电服服主的第一选择。
2. **Leaves**: 生电服务端，旨在修复Paper破坏的原版特性，而且掺入了钠 (Sodium)，自带假人部署功能。
3. **Lumina**: 是Folia的生电版本，修复了Folia破坏的原版特性，如果对于插件依赖较少的话可以一试，适用于人较多的生电服。

最后，建议在最终决定前尝试几种不同的服务端，评估它们在实际环境中的表现，这将帮助你做出最适合的选择。**记住，没有“最好”的服务端，只有最符合你需求的服务端。**

### 按服务器环境挑选
#### A. 多线程服务器/大型服务器（100人以上）
1. **Folia**: 它向服务器增加了区域化多线程功能，旨在解决 Minecraft 服务器单线程运行导致的性能瓶颈问题。
2. **Lumina**: 是Folia的生电版本，修复了Folia破坏的原版特性，如果对于插件依赖较少的话可以一试，适用于生电服。

#### B. 中型服务器（10~100人）
1. **Folia**：尽管你的服务器不是大型服务器，但它仍然是最优选择，解决了单线程服务器的瓶颈问题，是它成为最优选择的不二法门。
2. **Lumina**：没得说，生电服最佳选择。
3. **Paper**：单线程性能王，但是不建议放在第一选择上，备用（也是因为它破坏的原版特性比较多）。

## 如何更换服务端？
一般来说，我们可以使用“**直接替换法**”来替换服务端，过程如下：

1. 停止服务器（如果已经停止可以掠过这一步骤）
2. 下载所需要更换的服务端
3. 删掉原有的服务端
4. 上传你所需要更换的服务端
5. 把服务端JAR文件名和启动脚本对一下，检查是否有误，如有误就修正一下错误
6. 启动服务器
7. 检查控制台，看看是否有报错或出现任何问题
8. 完成！
