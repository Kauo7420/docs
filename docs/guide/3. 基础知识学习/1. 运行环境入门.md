## 运行环境概述
如果要思考这个问题，那么你可以先自行思考另一个问题：家园是不是人类赖以生存的地方？如果是，那么我们就会说**家园是人类的“生存环境”**。同理，如果你要架设一个Minecraft服务器，就必须要有一个运行环境。也就是说，**运行环境就是Minecraft服务器的“生存环境”**，同时也是服务器的前置（必需品）。

运行环境可以大致分为两个层面：

1. **软件层面**：一般代指Minecraft的前置软件，比如想要游玩Java版MC，你就必须要安装Java环境。
2. **硬件层面**：一般代指设备，即能承载Minecraft服务器的硬件运行环境，比如电脑、云服务器、虚拟主机。

## 运行环境分解
### 软件层面

让我们先从软件层面开始讲起吧，因为在一般情况下，选择软件不需要花费额外的费用。

这里设立了一个表格帮助大家快速浏览概要，以便接下来的详细详解阅读：

| 方面 | 描述 |
| --- | --- |
| Java | 若要运行一个Minecraft服务器，则Java环境必须安装，否则将无法正常运行服务器，而Java的版本选择也跟Minecraft的版本息息相关 |
| 前置插件 | 若服务器为插件服，则需注意部分插件可能需要安装前置插件以确保正常主插件的正常运行，你需要检查服务器所安装的插件是否都满足了前置插件需求 |
| 管理面板 | 若要方便、快捷、高效地管理一个服务器，那么管理面板是一个不二之选，它可以提供UI可视化操作管理服务器 |

#### 1. ==Java==——Minecraft游戏本体的必需品

人类的生存需要家园，那么Minecraft的生存就离不开Java。一般来说，我们在软件层面的必备品只有Java。在一般情况下，有了Java就能运行Java版Minecraft服务器，没有？那就想Peach。

根据[Minecraft中文Wiki](https://zh.minecraft.wiki/w/Java%E7%89%88#%E8%BD%AF%E4%BB%B6%E9%9C%80%E6%B1%82)的信息来看，在Java版本中，软件层面的要求如下：

| 版本 | Java版本需求 |
| --- | --- |
| 1.12以下 | Java |
| 1.12 (17w13a) ~ 1.16.5 | Java 8及以上版本 |
| 1.17 (21w19a) ~ 1.17.1 | Java 16及以上版本 |
| 1.18 (1.18-pre2) ~ 1.20.4 | Java 17及以上版本 |
| 1.20.5 (24w14a) ~ 1.21.11 | Java 21及以上版本 |
| 26.1 ~ Latest（最新版本） | Java 25及以上版本 |


如果你的服务器没有安装Java环境，那么你的服务器也将无法正常运行起来。

::: tip JRE和JDK的区别

**JRE(Java Runtime Enviroment)** 是 Java 的运行环境。面向 Java 程序的使用者，而不是开发者。如果你仅下载并安装了 JRE，那么你的系统只能运行 Java 程序。JRE 是运行 Java 程序所必须环境的集合，包含 JVM 标准实现及 Java 核心类库。它包括 Java 虚拟机、Java 平台核心类和支持文件。它不包含开发工具(编译器、调试器等)。

**JDK(Java Development Kit)** 又称 J2SDK(Java2 Software Development Kit)，是 Java 开发工具包，它提供了 Java 的开发环境(提供了编译器 javac 等工具，用于将 java 文件编译为 class 文件)和运行环境(提 供了 JVM 和 Runtime 辅助包，用于解析 class 文件使其得到运行)。如果你下载并安装了 JDK，那么你不仅可以开发 Java 程序，也同时拥有了运行 Java 程序的平台。JDK 是整个 Java 的核心，包括了 Java 运行环境(JRE)，一堆 Java 工具 tools.jar 和 Java 标准类库 (`rt.jar`)。

:::

#### 2. ==前置插件==——插件服的必需品

如果你的Minecraft服务器的类型是插件服，那么你的服务器肯定会和前置插件脱不了关系。

前置插件，即部分插件的依赖库或为附属关系的插件。部分插件的正常运行就需要依赖于前置插件，没有前置插件，部分插件就无法正常运行。也就是说，前置插件是部分插件的必需品。

如果要判断一个插件是否需要安装前置插件，则需要仔细阅读插件的相关说明，比如插件原帖或插件指南文档。

### 硬件层面

接下来是硬件层面，也是大多数服主所关心的地方，当然，这个层面也是最~~考验财力💰~~的层面。

这里设立了一个表格帮助大家快速浏览概要，以便接下来的详细详解阅读：

| 方面 | 描述 |
| --- | --- |
| CPU | 它是最核心、关键的部位，它主要负责处理复杂的计算任务，越高级的CPU可以承载越庞大的计算量，计算的方面可以是地图加载、玩家数据、红石机械、实体移动、TNT爆炸、粒子效果、方块状态更新、区块更新等等 |
| 内存 | 它是服务器承载能力的体现，其承载的内容为服务器运行状态时服务器所加载的数据，更大的内存能承载更多的服务器内容，比如能让更多的玩家流程在线服务器等等 |
| 磁盘 | 它是服务器储存容量的体现，服务器的基本数据都会保存在本地，更大的磁盘空间能储存更多的内容，比如地图存档、玩家数据、插件模组及其配置文件、日志文件等等 |

#### 1. ==CPU==——服务器的核心中坚力量

**CPU（Central Processing Unit），中文名为中央处理器**。有一定电脑常识的同学都知道，CPU 是整个电脑的核心所在，它是整台电脑的“大脑”。那么同理，我们也可以说它是服务器的“大脑”，因为大部分服务器数据都需要经 CPU 之手，才能正常返回显示给玩家看。

服务器内的游戏数据处理都需要通过 CPU 进行计算然后返回，比如发生了一次苦力怕爆炸时，就需要通过 CPU 计算一些数值，比如爆炸的破坏半径、造成的伤害等等，计算完之后再以数据包的形式发送给玩家。这就是一次CPU计算的一般流程。

CPU 的性能越强，那么它的处理速度也就越快，比如一个好的 CPU，它的地图生成速度要比一般的 CPU 更快。也就是说，这个“大脑”的计算能力被增强了，得到了强化。

一般来说，服主挑选服务器硬件的时候，比较看重的一个数值就是 CPU，因为一个强大且高效的 CPU 可以帮助服务器更快速地响应玩家的动作，减少延迟，并支持更多的玩家同时在线而不影响性能。

CPU的部分重要性能数值主要分为以下几类：

1. **核心数**：即 CPU 能同时执行任务的核心数量。这对于需要多核性能环境的服务器来说是一个很重要的数值，因为某些服务器需要 CPU 执行大量的并发操作，比如多世界、多插件，单核性能会在此刻败下阵来，多核的重要性就上来了。
2. **线程数**：一般指代在一个或多个 CPU 核心上能够同时处理的线程数量，更多的线程能够让服务器更有效地处理并发任务，让服务器拥有更快的响应速度。比如插件的事件处理、数据库访问、玩家生电机械的数据计算。但是，如果插件或软件没有对多线程做优化的话，那么多线程的优势就难以在此刻发挥出应有的光芒。
3. **时钟频率**：一般指代 CPU 每秒执行指令的次数，通常以 Hz(赫兹) 为单位表示，不过现在的 CPU 一般都以 GHz 为单位。时钟频率越高，每个核心在单位时间内可执行的命令数量越多，从而提高单线程性能。

#### 2. ==内存（RAM）==——服务器承载能力的外在体现

请各位同学注意一下，我们这里所说的内存，不是手机层面的储存内存，而是指运行内存，即 RAM。

**随机存取存储器（Random Access Memory，简称 RAM）**就是我们所要了解的东西。一般而言，RAM 是用来“临时”存放服务器资源数据的地方，在服务器方面，一般被用来加载地图存档区块、缓存玩家数据信息以及承载服务器本体的运行。

如果服务器有充足的内存，那么服务器因资源耗尽而导致的崩溃问题的出现概率就会大幅度减少。充足的内存能确保服务器稳定运行下去，承载服务器的运营，而且还能改善加载速度和响应时间。

如果服务器安装了大量的插件或是高分辨率（高清）的资源包（材质包），那么内存在此刻就显得至关重要。

---

#### 3. ==磁盘（Disk）==——服务器规模的集中体现

**磁盘（Disk）**是一个广泛的概念，它的包含方面比较大，在服务器层面一般代指服务器的限定容量大小，而并非指电脑方面的硬盘等硬件。

在不考虑其他因素（如CPU、内存）的影响下，你的服务器储存空间越大，那么服务器所能包含的内容就越多，比如存放更多的玩家数据、插件数据和地图存档数据。这能够在一定程度上扩大你的服务器规模。

磁盘的概念很好理解，你就把他看成一个盒子，磁盘空间越大，盒子就越大，能装进去的东西就越多。
