很多小白服主在新安装一个插件后，都会碰到这么一个情况：插件指令居然没有**权限（Permission）**使用！这是怎么回事！

## 什么是权限？
### 简介
权限是Minecraft服务器中的一个**重要功能**，它==规定了哪些功能哪些人可以使用，哪些人禁止使用==。

权限在服务器运营中起了很大的作用，如果没有权限的出现和配置，那么服务器的一些插件功能什么的，将无法正常使用。

### 权限节点
权限节点是权限的**具体体现**，权限节点通常很**分散**，且有很多个分支。它通常在`权限插件`中被使用。

举个例子，`quickshop`是一个权限节点，`quickshop.other`又是一个节点，而且是`quickshop`的分支，但是它还能再分，再往下就是`quickshop.other.destroy`。

==每一个权限节点规定着不同功能的使用权限==，例如，拥有`quickshop.use`权限节点的玩家便可以使用QuickShop插件的箱子商店功能。

权限在配置中一般以权限节点的形式出现，以下是一些分类：

1. **父节点**：权限节点最开头的那一部分，比如在`quickshop.use`中，`quickshop`就是父节点；
2. **子节点**：权限节点中开头那一段的后面的部分，比如在`quickshop.other.destroy`中，`other.destroy`就是子节点；
3. **通配符**：权限节点中指向对应权限节点中的所有子节点，符号为`*`，比如`quickshop`父节点的子节点包含`quickshop.unlimited`、`quickshop.setowner`、`quickshop.refill`，那么通配符*将会指向他们三个子节点，意思就是包含了他们三个权限节点所有的权限。

### 权限组（Group）
如果要配置一个群体（比如普通玩家、VIP玩家、服务器管理员、服主等）的权限节点的话，那么一个一个玩家去单独配置肯定是不行的!!那得配到什么时候啊!!，这个时候，**权限组**的作用就上来了。

一般来说，每个权限插件都有一个默认的权限组`default`，这个就是配置所有玩家的权限所使用的权限组。

还不明白？刚刚说`default`权限组指向所有玩家，那么这里假设一个情景问题，我想要给所有玩家一个QuickShop插件的箱子商店使用权限，那么我就可以给`default`权限组一个`quickshop.use`权限节点，在配置完之后，所有玩家便获得了QuickShop插件的箱子商店使用权限。

::: tip
如果你想要配置其他一些群体玩家的权限节点，那么你就需要创建更多的权限组。
:::

如果你要给服务器详细划分玩家们的权限组，那么以下是一些可以参考的权限组模板**（非必要）**

| 权限组名称 | 对应的权限分配情况 |
| :--- | :--- |
| Owner（服主） | 尊贵的服主大大，包含所有服务器的权限 |
| Admin（管理员） | 协助管理服务器的管理员，包含一些管理权限，比如`/ban`、`/gamemode`一些指令权限，但是不包含`/reload`、`/stop`、`/restart`一类的指令权限 |
| Newbie（萌新） | 新加入服务器的玩家，可以给一些观察者权限，但是不建议给一些基础权限，比如TPA权限 |
| Player（玩家） | 服务器的普通玩家，包含一些基础权限，比如TPA、随机传送权限 |


## 如何配置？
如果要配置权限节点和权限组一类的权限功能，那么你需要给你的服务器安装权限插件/模组，一般来说，权限插件/模组的最佳选项是`LuckPerms`/`GroupManager`。

给你的服务器安装权限插件后，再根据权限插件文档所给出的使用方法配置权限，这里不细说，自行查询相对应的插件文档教程。

::: note

如果你的服务器安装了`LuckPerms`插件/模组，那么你可以使用以下指令设置权限节点：

`/lp group <权限组> permission set <权限节点> [true/false]`在某权限组启用/禁用一个权限节点

`/lp group <权限组> permission unset <权限节点>`取消设置一个权限节点

`/lp editor`打开网页版的权限配置面板（**新手和懒人推荐！**）

:::
