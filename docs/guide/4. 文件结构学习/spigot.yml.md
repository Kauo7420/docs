## 概述
**Spigot.yml** 文件是 Spigot 服务器的核心配置文件，它允许你定制服务器的多种特性，以满足特定需求。此文件通常与 **bukkit.yml** 文件一起使用，你可以在服务器根目录找到它们。

## 配置项详解
### 1. 命令（Commands）
#### `tab-complete`
+ **默认值**: 0
+ **类型**: 整数
+ **描述**: 控制命令自动补全的触发长度。0 表示所有命令均可补全；负数禁用补全；正数代表需输入的字符数。

#### `send-namespaced`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 是否显示带有 `<插件>:<命令>` 格式的命令。

#### `silent-commandblock-console`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 是否静默命令方块的输出至控制台。

#### `log`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 是否记录玩家命令至日志。

#### `spam-exclusions`
+ **默认值**: ["/skill"]
+ **类型**: 列表
+ **描述**: 排除被服务器聊天过滤器视为垃圾信息的命令列表。

#### `replace-commands`
+ **默认值**: ["setblock", "summon", "testforblock"]
+ **类型**: 列表
+ **描述**: 禁用 Bukkit 实现的命令，启用原版行为。

### 2. 消息（Messages）
消息可以通过 `&` 符号添加颜色代码，并使用 `\n` 创建新行。

+ **默认值** 和 **类型** 视具体消息而定
+ **描述**: 自定义各种系统消息，如白名单拒绝、未知命令、服务器满员等。

### 3. 统计（Stats）
#### `disable-saving`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 是否禁用玩家统计和成就保存。

#### `forced-stats`
+ **默认值**: {}
+ **类型**: 映射
+ **描述**: 强制设置某些统计数值。

### 4. 成就（Advancements）
#### `disable-saving`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 是否禁用玩家成就保存。

#### `disabled`
+ **默认值**: []
+ **类型**: 列表
+ **描述**: 禁用的成就列表，防止在游戏内显示。

### 5. 主要未分组设置（Main, ungrouped settings）
#### `debug`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 控制日志级别。

#### `netty-threads`
+ **默认值**: 4
+ **类型**: 整数
+ **描述**: 控制网络处理线程数量。

#### `bungeecord`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 开启或关闭 BungeeCord 相关功能。

#### `timeout-time`
+ **默认值**: 60
+ **类型**: 整数
+ **描述**: 服务器无响应时长，超时将进行线程转储。

#### `restart-on-crash`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 是否自动重启服务器。

#### `restart-script`
+ **默认值**: ./start.sh
+ **类型**: 字符串（文件路径）
+ **描述**: 服务器启动脚本路径。

#### `late-bind`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 延迟玩家进入服务器直到所有插件加载完成。

#### `sample-count`
+ **默认值**: 12
+ **类型**: 整数
+ **描述**: 控制客户端服务器列表中显示的随机玩家样本数量。

#### `player-shuffle`
+ **默认值**: 0
+ **类型**: 整数
+ **描述**: 调整玩家在网络包处理中的位置。

#### `filter-creative-items`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 是否阻止创造模式玩家生成黑名单内的物品。

#### `user-cache-size`
+ **默认值**: 1000
+ **类型**: 整数
+ **描述**: 控制用户缓存的最大玩家数。

#### `save-user-cache-on-stop-only`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 只有服务器停止时才保存用户缓存数据。

#### `moved-wrongly-threshold`
+ **默认值**: 0.0625
+ **类型**: 浮点数
+ **描述**: 移动异常检查阈值。

#### `moved-too-quickly-multiplier`
+ **默认值**: 10
+ **类型**: 浮点数
+ **描述**: 最大移动速度的乘数。

#### `log-villager-deaths`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 是否记录村民死亡日志。

#### `item-dirty-ticks`
+ **默认值**: 20
+ **类型**: 整数
+ **描述**: 控制物品状态检查的间隔。

### 6. 按世界设置（Per-world settings）
这些设置可以为每个世界单独配置。

#### `below-zero-generation-in-existing-chunks`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 是否在升级到 1.18+ 版本时，在现有区块下方生成。

#### `verbose`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 是否显示每个世界的详细报告和配置。

#### `unload-frozen-chunks`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 实验性选项，是否卸载冻结的区块。

#### `simulation-distance`
+ **默认值**: 10
+ **类型**: 整数
+ **描述**: 控制实体、方块和流体更新的范围。

#### `view-distance`
+ **默认值**: 10
+ **类型**: 整数
+ **描述**: 控制围绕每位玩家加载的区块数量。不能高于15或低于1。减少可以减轻服务器负载，尤其是在大量玩家在线时。
+ **注意**: 对于1.14.4及更高版本，全局选项默认为“default”，此时服务器读取`server.properties`中的`view-distance`设置。此设置可用于每个世界的视距。

#### `merge-radius`
+ **子项**:
    - `exp`:
        * **默认值**: 3.0
        * **类型**: 浮点数
        * **描述**: 控制地面上经验值球合并的范围。
    - `item`:
        * **默认值**: 2.5
        * **类型**: 浮点数
        * **描述**: 控制地面上物品合并的范围。

#### `chunks-per-tick`
+ **默认值**: 650
+ **类型**: 整数
+ **描述**: 控制每刻执行生长更新的区块数量。降低此值会使生长变慢，但能节省资源。

#### `item-despawn-rate`
+ **默认值**: 6000
+ **类型**: 整数
+ **描述**: 控制地面上物品实体消失前所需的游戏刻数。降低此值会使物品更快消失。

#### `mob-spawn-range`
+ **默认值**: 6
+ **类型**: 整数
+ **描述**: 控制玩家周围怪物可能生成的区块半径。增加会使得怪物更稀少且分散。

#### `growth`
+ **子项**:
    - `cactus`, `melon`, `pumpkin`, `sapling`, `cane`, `mushroom`, `wheat`:
        * **默认值**: 100
        * **类型**: 整数
        * **描述**: 控制列出作物/资源的生长速度。默认值(100)表示“原版”速度。

#### `entity-activation-range`
+ **默认值**: (动物: 32, 怪物: 32, 其他: 16)
+ **类型**: 整数
+ **描述**: 控制实体变为“激活”的范围。外部的实体将以较低频率运行以避免服务器卡顿。

#### `tick-inactive-villagers`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 控制村民是否在超出激活范围时仍被激活。禁用可能导致村民交易恢复速度变慢。

#### `entity-tracking-range`
+ **默认值**: (显示: 128, 玩家: 48, 动物: 48, 怪物: 48, 其他: 32)
+ **类型**: 整数
+ **描述**: 控制实体变为“可见”或“追踪”的范围。降低可帮助客户端减少延迟。

#### `save-structure-info`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 控制Spigot是否保存结构信息。禁用可能会影响末影之眼定位要塞等功能。

#### `random-light-updates`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 控制服务器是否随机采样区块以验证并修复光照。

#### `nerf-spawner-mobs`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 启用后，怪物生成器产生的怪物不会有任何AI。

#### `zombie-aggressive-towards-villager`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 控制僵尸是否攻击村民。禁用可减少AI处理时间，从而减少服务器卡顿。

#### `enable-zombie-pigmen-portal-spawns`
+ **默认值**: true
+ **类型**: 布尔值
+ **描述**: 控制地狱传送门是否会随机生成猪灵。禁用不影响猪灵通过地狱传送门。

#### `max-entity-collisions`
+ **默认值**: 8
+ **类型**: 整数
+ **描述**: 单个实体每刻碰撞次数上限，降低对实体在1x1空间的影响。

#### `dragon-death-sound-radius`
+ **默认值**: 0
+ **类型**: 整数
+ **描述**: 控制末影龙死亡声音的传播半径。

#### `wither-spawn-sound-radius`
+ **默认值**: 0
+ **类型**: 整数
+ **描述**: 控制凋零生成声音的传播半径。

#### `max-bulk-chunks`
+ **默认值**: 10
+ **类型**: 整数
+ **描述**: 设置每包发送的区块数量。

#### `max-tick-time`
+ **默认值**: (tile: 50, entity: 50)
+ **类型**: 整数
+ **描述**: 控制(tile)实体操作消耗的时间上限（毫秒）。

#### `clear-tick-list`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 潜在防止Tick列表随时间增长，但引入多个问题。仅建议在完全理解后果的情况下使用。

#### `hopper-alt-ticking`
+ **默认值**: false
+ **类型**: 布尔值
+ **描述**: 动态调整漏斗的Tick行为，以模仿原版行为并提升性能。

#### `hopper-amount`
+ **默认值**: 1
+ **类型**: 整数
+ **描述**: 控制漏斗每次循环最多取出/放入的物品数量。

#### `seed-*`
+ **默认值**: `village: 10387312`, `feature: 14357617`, `monument: 10387313`, `slime: 987234911`
+ **类型**: 整数
+ **描述**: 控制不同世界元素的生成种子。推荐在生成世界前修改以保护种子秘密。

#### `hunger`
+ **默认值**:
    - jump-walk-exhaustion: 0.05
    - jump-sprint-exhaustion: 0.2
    - combat-exhaustion: 0.1
    - regen-exhaustion: 6.0
    - swim-multiplier: 0.01
    - sprint-multiplier: 0.1
    - other-multiplier: 0.0
+ **类型**: 浮点数
+ **描述**: 控制进行相应动作时消耗的饥饿值。

#### `hanging-tick-frequency`
+ **默认值**: 100
+ **类型**: 整数
+ **描述**: 控制悬挂实体（如画、物品架等）的Tick更新间隔。

#### `ticks-per`
+ **子项**:
    - `hopper-transfer`:
        * **默认值**: 8
        * **类型**: 整数
        * **描述**: 控制漏斗推拉物品之间的服务器刻数。值为8时，行为与原版一致。
    - `hopper-check`:
        * **默认值**: 1
        * **类型**: 整数
        * **描述**: 控制漏斗自上次尝试以来尝试推拉物品的服务器刻数。例如，值为8意味着空漏斗每8刻检查上方物品实体和上方的库存。值为0或1时，行为与原版一致。当`hopper-alt-ticking`设为true时，此选项被禁用。更改此值会因同步问题破坏大多数漏斗装置。
