world是服务器的存档文件夹，也是服务器的核心文件夹之一。

::: tip

在Paper及其分支服务器当中，world被拆散开来了，拆成了`world`、`world_nether`、`world_the_end`这三个文件夹，并且它们之间的数据也是分开存放到三个文件夹当中的。

:::

我们来逐步解析一下这些文件/文件夹的作用：

| advancements | 储存玩家进度数据的文件夹，一般被用来修改玩家进度数据 |
| :---: | :---: |
| data | 包含了一些数据包（不是datapack那种数据包），如资源、行为包等，它们可以修改游戏的行为和外观 |
| datapacks | 存放数据包的地方<br/>如果想启用指定数据包，你就需要把数据包文件丢这里面 |
| DIM1 | 末地世界的存档数据，删除会导致末地重置 |
| DIM-1 | 下界世界的存档数据，删除会导致下界重置 |
| entities | 放置实体的JSON文件，配置实体的血量、攻击、行为等 |
| playerdata | 存储每个玩家的个人游戏数据，如身上的物品和下线位置 |
| poi | 用于储存世界的兴趣点，使用[区域文件格式](https://zh.minecraft.wiki/w/%E5%8C%BA%E5%9F%9F%E6%96%87%E4%BB%B6%E6%A0%BC%E5%BC%8F)保存各个区块内的兴趣点数据 |
| region | 存放主世界地图区块信息文件的文件夹 |
| stats | 存放玩家游戏数据统计信息，例如玩家的成就和进度 |
| level.dat | 游戏世界的主数据文件，包含了世界的游戏规则、难度、游戏模式、生成设置等信息 |
| level.dat_old | 是`level.dat`的备份版本 |
| session.lock | 一个锁定文件，用于防止多个进程同时修改世界数据，确保数据的一致性 |
| uid.dat | 储存世界中所有实体的唯一标识符（Unique ID）的文件<br/>每个实体（包括玩家、动物和怪物）都有一个唯一的ID，以便游戏能够正确地跟踪它们 |
| forcedchunks.dat | 储存被强制加载的区块信息的文件 |


### Paper端及其分支
Paper服务端的存档文件基本上跟Vanilla服务端大相径庭，但是有几个新文件，是Vanilla没有的东西：

| paper-world.yml | Paper服务端中用于配置特定世界设置的 YAML 格式配置文件<br/>不过这玩意一般没用，会被`config/paper-world-defaults.yml`的设置选项给覆盖掉，除非你想独立配置该世界 |
| :---: | :---: |
